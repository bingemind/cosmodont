TECHNICAL REQUIREMENTS

IMPORTANT — this site will be maintained long term if the 
client signs. Write the code accordingly:

1. MAINTAINABILITY
   - All content (text, phone numbers, hours, doctor names, 
     services, review quotes) must be stored in a single 
     CONFIG object at the top of the JS section — clearly 
     commented so a non-developer can update it
   - CSS variables for ALL colors, fonts, spacing — 
     defined at :root level with clear comments
   - Each section wrapped in clearly commented HTML blocks: 
     <!-- ═══ SECTION: HERO ═══ --> etc.
   - No inline styles except for dynamic JS values
   - All images use placeholder divs with data-image-id 
     attributes and a comment explaining where to drop 
     the real image

2. PERFORMANCE
   - Single HTML file, no external JS libraries
   - Google Fonts loaded with display=swap
   - Images lazy-loaded (loading="lazy")
   - CSS animations use transform and opacity only 
     (no layout-triggering properties)

3. FUTURE-PROOFING
   - WhatsApp chat button (bottom right, green, 
     links to wa.me/917992074028) — 
     clearly marked as replaceable with GrahakBot later
   - Form action is a placeholder — comment says 
     "replace with your form handler or WhatsApp API"
   - Add a comment at the top of the file:
     "DEVELOPER NOTES: How to update this site" 
     with 5 bullet points explaining the CONFIG object, 
     CSS variables, section structure, image replacement, 
     and form integration

4. RESPONSIVENESS
   - Mobile-first CSS
   - Breakpoints: 768px (tablet), 480px (mobile)
   - Test hamburger menu, service grid (3→2→1 columns), 
     doctor cards (3→1), testimonials carousel
